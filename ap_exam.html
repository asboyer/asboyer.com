My AP Computer Science Review Guide

Introduction

As I begin to prepare for the upcoming AP Computer Science exam in May, I thought that writing this blog post would be creative way to review and share what I have learned over the course of this year in my class.

This post will take you through everything that I am reviewing to prepare for the exam. To preface, I am taking AP Computer Science A, which is taught in Java. I would also like to add that I do not claim myself to be an expert exam taker, or even a Java expert. I am just a student reviewing for an exam.

While Java isn't necessarily my favorite programming language, it teaches the basics of object-oriented programming, and programming in general, very well. While I did have programming experience, I had zero Java experience going into this school year. If I would say anything about the class itself, I would say that the focus wasn't as much on the actual syntax of the language, but on the fundamentals of programming. This is reflected in the exam as well. Most syntax errors on the exam don't even result in point deductions!

Your first Java program

Like any other programming language, our first program in Java must be to print Hello World! It's just tradition.

class HelloWorld {
public static void main(String[] args) {

System.out.println("Hello World!");

// the classic hello world program
// by the way, this is a comment

/* this is a multi-line
comment, written by Andrew Boyer */
}
}

Let's break down everything going on here. As it appears, it takes a bit more effort to get a Java program up and going, especially if you are coming from a scripting language like Python, where all you need is print("Hello World!"). For those following along, you can run this program in your terminal like this:

$ javac HelloWorld.java && java HelloWorld

This code will, of course, print Hello World! to the console. Please note that the name of your file must be HelloWorld.java if you're class name is HelloWorld. In other words, the name of your file must be the name of your class.

Okay, now we can get into what is really going on here, starting with class HelloWorld. This line is called a class definition. The line uses the keyword class to declare that a new class is being defined. A Class in Java is like an object constructor, which is the basis for everything in Java, because Java is an object-oriented programming language. The contents of the class is everything in between the corresponding curly braces { }. HelloWorld is an identifier that is the name of the class.

The next line of code in our program is called the main method. In Java, every program must contain this method. The main method is the "entry point" for the program, and this is the code that actually "runs" when we run the program. Code outside of the main method will not execute unless it is being called from inside the main method. But, this main method is a pretty complicated line of code:

public static void main(String[] args)

Let's go from left to right. The public is so that the Java Virtual Machine (JVM) can execute the method from anywhere. The specifics behind the JVM and how is works is enough for an entirely different blog post, so I will skip on the details for that. Next, we have static, which is there so that the main method can be called without an object. The void is there to specify that the method doesn't return anything. We have the main() because that is the name configured in the JVM, and String[] args is there because the main method accepts a single argument: an array of elements of type String (the name of the array is "args").

So, now we know what a Class Declaration is and why we need a Main Method. Let's move on to analyzing the actual code that is running.
